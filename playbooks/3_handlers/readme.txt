handlers выполняются в самом конце исполнения файла для тех задач, в которых прописан notify. 

Удобно тем, что выполняются не каждый раз при выполнении плейбука, а только при выполнении задачи, которая дергает handler. 

В этом примере файл nginx.conf будет передан на сервер только в том случае, если бы изменен. Поэтому при повторном запуске, если файл не был изменен, задача update nginx config не будет выполнена снова, а потому и handler `restart nginx` не будет выполнен. 

Результатом выполнения playbook-a будет доступ к 127.0.0.1:8080
